¿Qué es Git?

Git es un popular sistema de control de versiones. Fue creado por Linus Torvalds en 2005, y ha sido mantenido por Junio Hamano desde entonces.

Se utiliza para:

  1. Seguimiento de cambios de código
  2. Seguimiento de quién hizo los cambios
  3. Codificación de colaboración

¿Qué hace Git?

  1. Gestionar proyectos con Repositorios
  2. Clonar un proyecto para trabajar en una copia local
  3. Controle y rastree los cambios con Estadificación y Comprometerse
  4. Sucursal y Funsionar para permitir trabajar en diferentes partes y versiones de un proyecto
  5. Tire la última versión del proyecto a una copia local
  6. Empujar actualizaciones locales del proyecto principal

¿Por qué Git?

  1. ¡Más del 70% de los desarrolladores usan Git!
  2. Los desarrolladores pueden trabajar juntos desde cualquier parte del mundo.
  3. Los desarrolladores pueden ver la historia completa del proyecto.
  4. Los desarrolladores pueden volver a versiones anteriores de un proyecto.

Trabajando con Git

  1. Inicializa Git en una carpeta, convirtiéndolo en un Repositorio
  2. Git ahora crea una carpeta oculta para realizar un seguimiento de los cambios en esa carpeta
  3. Cuando se cambia, agrega o elimina un archivo, se considera modificado
  4. Selecciona los archivos modificados que desea Etapa
  5. El Etapa los archivos son Comprometido, que pide a Git que almacene un permanente instantánea de los archivos
  6. Git le permite ver la historia completa de cada compromiso.
  7. Puede volver a cualquier confirmación anterior.
  8. Git no almacena una copia separada de cada archivo en cada commit, pero ¡realiza un seguimiento de los cambios realizados en cada confirmación!
